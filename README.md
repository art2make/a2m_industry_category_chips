# Industry Category Chips для OpenCart 3 / ocStore 3

Версия: `1.0.7`

Модуль добавляет в карточку категории вкладку `Отраслевые категории`. В ней можно выбрать любые существующие категории, которые будут показаны сверху на странице текущей категории как навигационные чипсы/плитки.

## Что исправлено в 1.0.7

Замена блока чипсов переведена на один универсальный regex: от `$data['categories'] = array();` до `foreach ($results as $result)`. Это работает и после удаления старого условия `$show_subcategories_for`, и с обёртками FilterVier / UniShop2. Если отраслевые категории не заданы, блок автоматически показывает обычные подкатегории.

## Что исправлено в 1.0.6

Замена блока подкатегорий в витринном контроллере переведена на `regex="true"`. Стандартный OCMOD ищет совпадения построчно, поэтому многострочный `<search>` из версий `1.0.2`-`1.0.5` фактически не применялся, хотя данные в админке сохранялись.

## Что исправлено в 1.0.5

Отраслевые категории больше не зависят от условий FilterVier `$url_plus` / `$view_child_cat`. Эти условия применимы к стандартным подкатегориям и фильтрованной выдаче, но не к управляемому отраслевому блоку.

## Что исправлено в 1.0.4

Отраслевые категории больше не зависят от настройки UniShop2 `Отключить показ подкатегорий`. Эта настройка относится к штатным подкатегориям, а отраслевые чипсы должны выводиться как отдельная навигация.

## Что исправлено в 1.0.3

Добавлена отдельная OCMOD-операция для проектов, где блок подкатегорий уже изменен модулем FilterVier. Без этого витринный контроллер мог оставаться на старой логике `$show_subcategories_for`, хотя данные в таблице сохранялись.

## Что исправлено в 1.0.2

В версии `1.0.2` модификатор заменяет весь старый блок вывода подкатегорий с `$show_subcategories_for`, а не одну строку `getCategories()`. Это нужно для проектов, где ранее уже был временный список ID категорий.

В версии `1.0.1` были упрощены точки поиска для модели категорий, чтобы метод `getIndustryCategories()` гарантированно попадал в `admin/model/catalog/category.php`.

## Установка

1. Загрузите `industry_category_chips.ocmod.zip` через `Расширения -> Установка расширений`.
2. Откройте `Расширения -> Модификаторы`.
3. Нажмите `Обновить`.
4. Очистите кеш темы/модификаторов.
5. Проверьте журнал модификаторов на ошибки `NOT FOUND`.

Таблица создается автоматически при первом сохранении категории или открытии страницы категории. Для ручной установки используйте `sql/install.sql`, заменив `{DB_PREFIX}` на префикс базы.

## Использование

1. Откройте `Каталог -> Категории`.
2. Отредактируйте категорию.
3. Перейдите во вкладку `Отраслевые категории`.
4. Добавьте нужные категории через autocomplete.
5. При необходимости задайте сортировку.
6. Сохраните категорию.

## Удаление

1. Удалите OCMOD `Industry Category Chips`.
2. Обновите модификаторы.
3. Если нужно удалить данные связей, выполните `sql/uninstall.sql`.
