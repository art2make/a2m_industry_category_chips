# a2m - Industry Category Chips для OpenCart 3 / ocStore 3

Модуль добавляет в карточку категории отдельную вкладку `Отраслевые категории`. В ней можно выбрать любые существующие категории, которые будут показаны сверху на странице текущей категории как навигационные чипсы/плитки.

Модуль не меняет дерево категорий, `parent_id`, `category_path` и привязки товаров. Он хранит только связи вида `категория страницы -> категории для вывода`.

## Состав пакета

- `install.xml` - OCMOD-модификатор для установки через стандартный установщик OpenCart.
- `sql/install.sql` - SQL для ручного создания таблицы, если нужно выполнить миграцию заранее.
- `sql/uninstall.sql` - SQL для полного удаления таблицы с данными связей.
- `docs/usage.md` - инструкция для контент-менеджера.
- `docs/developer-notes.md` - технические заметки по интеграции.

## Установка через OpenCart

1. Упакуйте содержимое этой папки в ZIP так, чтобы `install.xml` лежал в корне архива.
2. Назовите архив, например: `industry_category_chips.ocmod.zip`.
3. В админке откройте `Расширения -> Установка расширений`.
4. Загрузите архив.
5. Откройте `Расширения -> Модификаторы` и нажмите `Обновить`.
6. Очистите кеш темы/модификаторов, если он включен на проекте.

Таблица `category_industry_category` создается автоматически при первом сохранении категории или открытии страницы категории. Если нужно создать ее заранее, выполните `sql/install.sql`, заменив `{DB_PREFIX}` на префикс базы проекта.

## Проверка после установки

1. Откройте `Каталог -> Категории`.
2. Отредактируйте нужную категорию.
3. Перейдите во вкладку `Отраслевые категории`.
4. Через autocomplete добавьте несколько категорий и сохраните.
5. Откройте эту категорию на витрине.
6. Вверху каталога должен появиться стандартный блок UniShop2 `category-list` с выбранными категориями.

## Совместимость

Пакет рассчитан на OpenCart 3 / ocStore 3 и тему UniShop2, где страница категории использует стандартный массив `$data['categories']` в `catalog/controller/product/category.php`.

Если на проекте уже есть другие OCMOD-модификаторы страницы категории, фильтра, шаблона или админской формы, после установки обязательно проверьте журнал модификаторов. При конфликте нужно адаптировать `install.xml` под конкретные поисковые фрагменты проекта.

## Удаление

1. Удалите модификатор `Industry Category Chips` в админке.
2. Обновите модификаторы.
3. При необходимости удалите таблицу связей через `sql/uninstall.sql`, заменив `{DB_PREFIX}` на префикс базы.
